# 데이터 추적성 UI 유튜브/블로그 확장 초고

## 1. 콘텐츠 컨셉
- 이번 PoC 개선의 핵심은 `숫자를 보여주는 화면`에서 `숫자의 근거를 끝까지 추적하는 화면`으로 바꾼 것이다.
- 한 문장으로 정리하면, 모든 숫자에 근거 영수증을 붙이는 UI를 구현한 사례다.

## 2. 제목 후보
1. 왜 이 숫자인지 끝까지 추적되는 대시보드 만들기 (L1→L2→L3)
2. AI 대시보드 신뢰성 높이는 법: 결과 뒤의 원본까지 연결하기
3. 출처 없는 숫자 금지: Traceable UI 실전 적용기
4. PoC를 제품처럼 바꾸는 방법: 데이터 추적성 UX 구축
5. 숫자 검증 가능한 화면 설계: History·Source Diff까지 구현

## 3. 오프닝 멘트(30초)
대시보드 숫자가 의심스러울 때, 보통은 엑셀·슬랙·DB를 오가며 근거를 찾는다.  
이번 화면은 그 과정을 UI 안으로 가져왔다.  
숫자 하나를 클릭하면 계산식이 나오고, 원본 파일이 나오고, 이전 버전과의 차이까지 한 번에 비교된다.  
오늘은 이걸 PoC에 어떻게 적용했는지 비개발자도 이해할 수 있게 보여준다.

## 4. 문제 정의
1. 기존 화면은 숫자는 빠르게 보이지만 “왜 그 숫자인지” 확인하기 어려웠다.
2. AI 값과 사람 값이 섞여 있을 때 신뢰 이슈가 생겼다.
3. 이상치가 나오면 근거를 찾는 시간이 길었다.
4. 버전이 바뀌면 숫자 변화 원인을 설명하기 어려웠다.

## 5. 해결 원칙
- 원칙: `No Orphaned Data`
- 화면에 표시되는 모든 값은 아래 3단계로 연결된다.
1. `L1 Surface`: 결과값
2. `L2 Logic`: 계산 근거/설명
3. `L3 Source`: 원본 파일/DB/외부 링크

## 6. 실제 구현 기능
1. 모든 핵심 값에 `출처` 인디케이터를 추가했다.
2. 값 클릭 시 오른쪽 `TracePanel`이 열리도록 구성했다.
3. 패널에 `Summary`, `Logic`, `Sources` 탭을 제공한다.
4. `Sources`에서 원본 링크와 하이라이트 정보를 확인할 수 있다.
5. AI/Human 배지로 데이터 생성 주체를 분리했다.
6. `History` 탭으로 버전별 비교를 지원한다.
7. 값 변화량을 색상으로 표시한다. 증가=녹색, 감소=적색.
8. 신뢰도 변화량을 `pp` 단위로 표시한다.
9. source diff를 `추가/제거/공통`으로 보여준다.
10. diff 항목 클릭 시 `Sources` 탭으로 이동해 해당 source를 바로 포커싱한다.
11. 버전 추이 스파크라인을 제공해 값 흐름을 직관적으로 보여준다.
12. CCTV trace는 `Frame/Pipeline/Raw` 전용 탭으로 분리했다.
13. Forecast 차트 툴팁에서도 바로 trace 패널 진입이 가능하다.

## 7. 비개발자 데모 시나리오
예시: `Est. Weight 1,081g`이 의심스러운 상황
1. 값 옆 `출처` 클릭
2. `Summary`에서 현재 값, 버전, 스냅샷 시간 확인
3. `Logic`에서 계산식 확인
4. `Sources`에서 원본 DB/파일 확인
5. `History`에서 과거 버전과 비교
6. source diff 확인
7. 바뀐 source를 눌러 `Sources` 탭에서 상세 근거 확인

## 8. 쉬운 비유
1. L1은 성적표
2. L2는 풀이 과정
3. L3는 시험지 원본
4. History는 지난 시험과의 비교표

## 9. 데이터 구조 설명(간단)
값만 보내지 않고, 로직과 원본과 버전 히스토리를 함께 보내 UI에서 설명 가능성을 보장한다.

예시 필드 구성
1. `display_value: "82%"`
2. `logic_summary: "전월 대비 12% 상승"`
3. `data_source: [{ type, name, url, page, highlight_text }]`
4. `is_ai_generated: true`
5. `version_history: [{ source_version, snapshot_at, display_value, confidence, data_source }]`

## 10. UX 포인트
1. Side Panel 중심으로 컨텍스트를 유지한다.
2. 단계적 정보 노출로 인지 부하를 줄인다.
3. 클릭 2~3번 이내로 근거 도달이 가능하다.
4. 설명 가능성을 UI 컴포넌트 레벨에서 강제한다.

## 11. 기대 효과
1. 숫자 설명 시간 단축
2. AI 결과 수용성 개선
3. 이슈 대응 근거 전달 속도 향상
4. 보고/회의에서 설명 품질 향상

## 12. 검증 결과
1. lint 통과
2. build 통과
3. 문서 동기화 완료

## 13. 마무리 멘트
대시보드가 신뢰를 잃는 순간은 숫자가 틀려서가 아니라, 숫자를 설명할 수 없을 때다.  
이번 개선은 정확도 자체보다 `설명 가능성`을 제품 기능으로 만든 사례다.  
다음 단계로는 추적 체인 공유 링크와 리포트 자동화까지 확장할 수 있다.
