# Palantir Homepage Main Design Spec

Status: reference  
Observed on: 2026-03-12  
Source: `https://www.palantir.com/`  
Viewport baseline: desktop `1440px` wide  
Goal: 현재 Palantir 홈의 메인 페이지를 해석해, 이 문서만으로도 높은 확률로 같은 계열의 UI를 재구성할 수 있게 만드는 설계 기준서

## 1. 문서 목적

이 문서는 Palantir 홈의 `카피`를 옮기는 문서가 아니다.  
이 문서는 Palantir 홈의 `레이아웃 구조`, `시각적 위계`, `컴포넌트 문법`, `밀도`, `간격`, `표면 처리`, `타이포 운용법`을 재현하기 위한 구현 기준서다.

이 문서의 목표는 아래 두 가지다.

- 다른 브랜드 콘텐츠를 넣어도 Palantir 계열의 첫인상과 구조적 리듬이 나오게 한다.
- 구현자가 별도 디자인 파일 없이도 화면 구조를 일관되게 다시 만들 수 있게 한다.

## 2. 핵심 디자인 테제

Palantir 홈의 인상은 화려함보다 `구조적 권위감`에서 나온다.

- 첫 화면은 설명보다 `대형 장면`이 먼저 온다.
- 텍스트는 많지 않지만, 각 텍스트는 매우 크고 단정하다.
- 큰 이미지를 쓰되, 색은 절제하고 회색 계열의 산업적 표면 위에 올린다.
- 섹션은 길게 설명하지 않고, `짧은 모듈`을 많이 배치해 훑게 만든다.
- 카드, 버튼, 링크는 거의 장식이 없고, 아주 얇은 선과 작은 반경만 사용한다.
- 시각 리듬은 `Hero -> Tab Rail -> 큰 문장 -> 모듈 보드 -> 제품 보드 -> 선언형 블록 -> 증거/인용 -> CTA -> 푸터`로 이어진다.

한 줄로 요약하면:

`미니멀한 산업용 에디토리얼 + 대형 미디어 무대 + 짧은 정보 모듈의 반복`

## 3. 전역 레이아웃 시스템

### 3.1 페이지 골격

- 상단에는 얇은 안내 바와 헤더가 겹친다.
- 첫 화면은 `거의 풀블리드`로 느껴지는 Hero다.
- 그 아래부터는 `넓은 중앙 컨테이너` 안에서 모듈이 이어진다.
- 관찰 기준 데스크톱 1440px에서 실질 콘텐츠 폭은 약 `1360px` 수준으로 보인다.
- 좌우 여백은 약 `40px` 안팎이다.
- 섹션 간 전환은 배경색, 이미지 밀도, 카드 구조 변화로 만든다. 장식용 디바이더에 의존하지 않는다.

### 3.2 컨테이너 규칙

권장값:

- `--page-max: 1360px`
- `--page-gutter: clamp(20px, 2.8vw, 40px)`
- `--section-gap-y: clamp(72px, 8vw, 120px)`

구현 원칙:

- Hero만큼은 `컨테이너 안에 갇힌 느낌`이 나면 안 된다.
- Hero 내부 오브젝트는 화면 가장자리 가까이까지 커질 수 있어야 한다.
- Hero 아래 본문은 반드시 `넓고 낮은` 레이아웃이어야 한다. 세로로 길고 좁은 카드 중심 구조로 만들면 느낌이 무너진다.

## 4. 색과 표면 시스템

현재 홈 기준으로 읽히는 표면 구조는 아래와 같다.

### 4.1 핵심 색상 토큰

아래 값은 구현용 근사치다.

- `--bg-page: #ffffff`
- `--bg-hero: #d8d8d6`
- `--bg-hero-deep: #bdbdb8`
- `--bg-surface: #f3f3f1`
- `--bg-surface-alt: #ececea`
- `--bg-dark: #202426`
- `--bg-dark-soft: #2a2f33`
- `--text-strong: #1e2124`
- `--text-body: #2f3338`
- `--text-muted: #6f7479`
- `--text-soft: #aaaaaa`
- `--text-on-hero: #efefef`
- `--border-light: #dfdfdf`
- `--border-mid: #cfcfcf`

### 4.2 표면 운용 규칙

- 큰 배경은 흰색 하나로 끝내지 않는다. `밝은 회색`, `웜그레이`, `산업용 금속 톤`을 섞는다.
- 다만 강한 그라디언트나 다색 배경은 쓰지 않는다.
- 흰 카드 위 카드, 회색 위 흰 카드, 검정 위 밝은 버튼의 대비가 핵심이다.
- 그림자보다 `면 대비`와 `1px 경계선`으로 구분한다.

### 4.3 금지 규칙

- 포화된 브랜드 컬러 대면적 사용 금지
- 두꺼운 그림자 금지
- 12px 이상 과한 라운드 금지
- 글래스모피즘 금지
- 네온/사이버 펑크식 광원 금지

## 5. 타이포 시스템

### 5.1 폰트 성격

관찰된 폰트 계열:

- 본문/일반 UI: `Alliance No.1` 계열
- Hero 대제목: `Alliance No.2` 계열

재현 원칙:

- 산세리프이되 흔한 SaaS 느낌이 아니라, 약간 문서적이고 출판물 같은 표정을 가져야 한다.
- Hero와 일반 본문을 같은 폰트로 처리하지 말고, Hero 쪽은 더 조형적인 패밀리를 써야 한다.

### 5.2 크기 체계

관찰값 기반 권장치:

- Hero H1: `80 / 78 / -3.4px / 400`
- 대형 브랜드 문장: `68 / 81.6 / -2.7px / 400`
- 섹션 타이틀 H2: `32~40 / 1.05~1.15 / 400`
- 카드 제목 H3: `26~32 / 1.1~1.2 / 400`
- 일반 본문: `18 / 1.5~1.7 / 400`
- 소형 메타/탭/링크: `16~18 / 1.2~1.4 / 400`

### 5.3 타이포 원칙

- 글자수는 짧게, 크기는 크게.
- Bold로 해결하지 않는다. 크기와 여백으로 위계를 만든다.
- 자간은 약간 좁힌다. 특히 큰 제목은 타이트해야 한다.
- 본문은 길어도 2~3줄에서 끊는다.
- 섹션마다 긴 설명문을 두지 않는다.

## 6. 간격, 선, 반경

### 6.1 반경

- 기본 카드 반경: `4px`
- 소형 탭/버튼 반경: `2px`
- 과장된 둥근 모서리 금지

### 6.2 경계선

- 거의 모든 구분은 `1px solid var(--border-light)` 정도로 정리한다.
- 섹션 경계보다 카드, 버튼, 리스트 항목 경계가 더 중요하다.

### 6.3 패딩

권장치:

- 탭 버튼: `12px 10~16px`
- 카드 내부: `24~32px`
- 큰 모듈 패널: `32~40px`
- Hero 상하 패딩: `40px` 기반

## 7. 섹션별 레이아웃 청사진

### 7.1 상단 안내 바

역할:

- 페이지 맨 위에 짧은 공지나 링크를 노출한다.
- 닫기 버튼이 있으며, 헤더보다 더 얇고 가볍다.

구성:

- 단일 행
- 좌측 정렬 텍스트 혹은 짧은 링크
- 우측 닫기 버튼

스타일:

- 어두운 텍스트
- 얇은 높이
- 배경은 투명 또는 매우 옅은 회색

### 7.2 헤더

역할:

- 브랜딩은 작게 유지하고, 행동 버튼은 우측에 단정하게 둔다.
- 메가 메뉴와 검색을 열기 위한 진입점 역할을 한다.

구성:

- 좌측 로고
- 우측 작은 CTA
- 검색 아이콘
- 메뉴 토글

스타일:

- 헤더 자체는 주연이 아니다.
- hero 위에 가볍게 얹힌 얇은 바처럼 보여야 한다.
- 버튼은 작고 단정하며, 과한 컬러를 쓰지 않는다.

### 7.3 Hero

이 페이지의 가장 중요한 구간이다.

구조:

- 화면 상단을 크게 점유하는 풀블리드형 장면
- 대형 미디어 오브젝트 1개가 왼쪽 또는 중앙을 크게 차지
- 큰 헤드라인은 이미지와 겹쳐 보이되 읽히게 배치
- 하단에는 `Scroll to Explore` 성격의 아주 작은 큐

스타일 규칙:

- 배경은 밝은 회색 계열
- 이미지 오브젝트는 공업 제품, 장비, 시스템 인터페이스, 기계적 렌더 등 `산업적 신뢰감`을 주는 것
- Hero에서 카드 여러 장을 띄우지 않는다. `하나의 장면`이 우선이다.
- Hero 카피는 5단어 안팎의 짧은 선언형 문장 구조가 적합하다.

금지:

- 텍스트 블록 + 설명문 + 보조 카드 3개 같은 SaaS hero
- 모션 과잉
- 다색 배경

### 7.4 탭형 캠페인 레일

Hero 다음 핵심 장치다.

구조:

- 상단에 짧은 카테고리 버튼열
- 우측에 `See All` 링크
- 하단에 가로 스크롤 가능한 대형 카드 레일

탭 규칙:

- 탭은 텍스트만 있거나 아주 미세한 배경면만 가진다.
- 활성 탭은 흰 배경과 더 짙은 텍스트로 드러낸다.
- 탭 수는 많아도 된다. 대신 한 번에 모두 읽히지 않아도 된다.

레일 카드 규칙:

- 카드 폭은 커야 한다. 작은 썸네일 카드가 아니라 `캠페인 패널`처럼 보여야 한다.
- 카드 상단 대부분은 이미지가 차지한다.
- 하단에는 작은 라벨과 1줄 제목만 둔다.
- 버튼은 별도 CTA 버튼보다 `화살표`, `↗`, `→` 같은 짧은 신호로 끝낸다.
- 카드 여러 장이 한 번에 보이되, 정확히 균등 그리드로 보일 필요는 없다.

권장 구현값:

- 카드 폭: `420~520px`
- 카드 이미지 비중: 전체 높이의 `65~75%`
- 카드 간격: `16~20px`

### 7.5 대형 브랜드 문장 섹션

역할:

- Hero와 카드 레일 다음에 브랜드의 한 문장을 큰 활자로 박아 넣는다.

구조:

- 텍스트만 거의 단독으로 존재
- 중앙 정렬
- 최대 폭은 넓게 허용

스타일:

- 60px 이상 큰 활자
- 줄 수는 2~4줄
- 배경은 흰색 혹은 매우 옅은 회색

이 섹션의 목적은 설명이 아니라 `정지와 호흡`이다.

### 7.6 모듈 보드: Latest News / Offerings / Impact / Quick Links

이 페이지의 중단부 문법을 정의하는 핵심 구간이다.

구조:

- 2열 또는 4분할 모듈 보드
- 각 모듈은 독립된 정보 블록
- 모듈 제목은 짧고, 내용은 1~2개 항목만 보여준다

모듈 유형:

- 뉴스 모듈: 메타 + 썸네일 + 제목 + 짧은 설명
- Offerings 모듈: 설명형 카드 한 개
- Impact 모듈: 사례 카드 한 개
- Quick links 모듈: 단순 텍스트 링크 리스트

스타일:

- 흰 바탕
- 얇은 경계선
- 좌우 컬럼 균형이 중요
- 모듈 내부 요소는 서로 다른 타입이어도 `패딩`, `제목 크기`, `경계선 굵기`를 공유해야 한다

금지:

- 네 개 모듈을 모두 같은 카드 스타일로 통일하는 것
- 모든 모듈에 설명문을 많이 넣는 것

### 7.7 제품/플랫폼 카드 스트립

관찰상 `Our Software` 같은 제목 아래 5개 내외의 세로형 카드가 가로로 정렬된다.

구조:

- 섹션 제목 1개
- 하단에 4~5개 카드
- 각 카드는 짧은 이름, 한두 줄 설명, 작은 화살표 CTA

스타일:

- 카드 배경은 흰색 또는 아주 옅은 회색
- 높이는 균일
- 카드 간격은 좁고, 폭은 비슷하게

이 구간은 정보 전달보다 `카탈로그 리듬` 형성이 목적이다.

### 7.8 선언형 분할 블록

스크린샷 기준으로 이미지 패널과 텍스트 패널이 반반에 가깝게 배치된다.

구조:

- 좌측: 사진 또는 실제 현장 이미지
- 우측: 대형 문장 + 짧은 서브카피 + 작은 링크

스타일:

- 흰 배경
- 좌우 분할
- 텍스트 패널도 카드처럼 과하게 장식하지 않는다
- 메시지는 짧고 선언적으로 간다

### 7.9 증거/인용 그리드

이 구간이 Palantir 특유의 `밀도`를 만든다.

구조:

- 섹션 제목 1개
- 그 아래 많은 수의 짧은 인용 카드
- 카드마다 회사명 또는 조직명 + 한 단락 인용

스타일:

- 작은 카드가 다수 배치됨
- 카드 배경은 밝은 회색 계열
- 텍스트는 작지만 빽빽하다
- 카드 수가 많아야 한다. 3개 정도로 끝내면 느낌이 안 난다.

중요 원칙:

- 이 구간은 `브랜드 자랑`이 아니라 `사회적 증거의 압력`을 만드는 섹션이다.
- 카드 수와 반복이 중요하다.

### 7.10 하단 CTA

구조:

- 2개 액션을 나란히 둔다
- 좌측은 밝은 CTA
- 우측은 어두운 CTA

스타일:

- 버튼이 아니라 큰 액션 블록처럼 보인다
- 텍스트는 짧다
- 화살표는 항상 포함

권장 구현:

- 높이 `64px` 이상
- 좌측 밝은 면 / 우측 어두운 면
- 둘 다 동일한 폭으로 맞춘다

### 7.11 푸터

구조:

- 좌측: 저작권, 지역 선택, 소셜
- 우측: 4열 이상의 대형 링크 컬럼

스타일:

- 매우 정보 밀도가 높다
- 글자 크기는 작다
- 정리된 표처럼 보여야 한다

푸터에서 중요한 것은 감성이 아니라 `문서실 같은 정돈감`이다.

## 8. 이미지 사용 규칙

- 이미지가 많아도 전체 팔레트는 차갑고 절제돼야 한다.
- 사진은 광고 사진보다 `현장`, `장비`, `제품`, `산업 시스템`, `회의`, `운영 현장`이 어울린다.
- 제품 이미지는 UI 전체를 친절히 보여주기보다 크롭된 장면이 더 적합하다.
- 색이 튀는 이미지는 카드 하나 정도까지만 허용하고, 전체 톤은 회색/청색/금속색에 묶어야 한다.

## 9. 인터랙션 원칙

- hover는 이동보다 `색/명도 변화` 위주
- 카드 hover는 살짝 떠오르기보다 `화살표`나 `라벨`이 더 선명해지는 정도면 충분
- 탭 전환은 즉각적이어야 한다
- 수평 레일은 스크롤과 버튼 두 방식 모두 허용
- 전체 모션은 조용해야 한다. 흥분된 느낌이 나면 실패다.

## 10. 복제 시 절대 유지해야 할 것

- Hero는 반드시 `하나의 큰 장면`이어야 한다.
- Hero 다음에는 반드시 `탭형 레일`이 와야 한다.
- 그 다음에는 반드시 `대형 브랜드 문장`이 와야 한다.
- 중단부는 `짧은 정보 모듈`로 구성해야 한다.
- 하단에는 `인용/증거 카드의 반복`이 있어야 한다.
- CTA는 2개로 단순화해야 한다.
- 전체 톤은 `밝은 산업용 에디토리얼`이어야 한다.

## 11. 복제 시 버려야 할 것

- 영웅 섹션 안에 너무 많은 카드/배지/지표를 넣는 것
- 검은 배경 위 네온 포인트를 남발하는 것
- 제품 소개를 긴 설명 카드 3개로만 처리하는 것
- 라운드가 큰 현대적 SaaS 카드 문법
- 얇고 긴 모바일형 카드가 데스크톱까지 그대로 오는 구조

## 12. 구현 체크리스트

- Hero가 한 화면에서 `장면 우선`으로 보이는가
- Hero 아래 탭 레일이 실제로 가로 흐름을 만드는가
- 큰 브랜드 문장이 충분히 큰가
- 중단부가 긴 설명 섹션이 아니라 짧은 모듈로 읽히는가
- 하단 증거 섹션이 카드 수와 밀도로 압박감을 주는가
- CTA가 두 개만 남아 있는가
- 전체 컬러가 밝은 회색/백색/차콜 범위 안에 머무는가
- 카드 반경이 작고 경계선이 얇은가
- 그림자 없이도 구조가 읽히는가

## 13. 이 문서를 실제 제작에 쓸 때의 해석 원칙

- 그대로 베끼지 말고 `구조와 문법`을 옮긴다.
- 카피와 산업 맥락은 교체해도 된다.
- 그러나 `Hero -> Rail -> Statement -> Modules -> Software -> Split Manifesto -> Proof Grid -> CTA -> Footer` 순서를 무너뜨리면 결과물의 느낌도 무너진다.
- Palantir의 느낌은 개별 컴포넌트보다 `배치 순서와 밀도`에서 온다.

## 14. 재현 우선순위

새 사이트를 만들 때 우선순위는 아래 순서로 둔다.

1. 전체 섹션 순서
2. Hero의 장면성
3. 탭형 가로 레일
4. 큰 문장 섹션
5. 중단부 모듈 보드
6. 증거 카드 수와 밀도
7. 색/타이포 미세조정

이 순서를 뒤집으면 비슷한 컴포넌트를 써도 Palantir 같은 느낌이 잘 나오지 않는다.
