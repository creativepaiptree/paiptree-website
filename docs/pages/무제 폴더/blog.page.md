# /blog 페이지 운영 문서

## 1. 문서 정보
- 라우트: `/blog`
- 페이지 파일: `src/app/blog/page.tsx`
- 문서 파일: `docs/pages/blog.page.md`

## 2. 목적
- 공개 사이트용 인사이트/아카이브 허브로 블로그 글 목록을 제공한다.
- 카테고리와 태그를 실제 필터로 사용해 탐색성을 높인다.
- 시각 스타일은 `docs/web-design-system.md` 기준을 따라 다크 헤더/인트로와 라이트 본문 쉘을 조합한다.

## 3. 화면/기능 구성
- 상단: `BlogHeader`
- 본문:
  - 다크 고정 카테고리 바
  - 다크 인트로 + 현재 필터 상태
  - 라이트 포스트 목록(좌측 메인 컬럼)
  - 라이트 소개/컬렉션/태그/빠른 링크(우측 사이드바)
- 하단: `Footer`

## 4. 데이터/상태
- `blogPosts`, 카테고리, 태그, 우측 보조영역 데이터는 정적 상수 사용
- 날짜는 언어 상태에 따라 `ko-KR` 또는 `en-US` 포맷으로 렌더링
- 클라이언트 상태: `selectedCategory`, `selectedTag`

## 5. 인터랙션 규칙
- 카테고리 pill 클릭 시 목록이 즉시 필터링된다.
- 카드 내부 태그 클릭 시 태그 필터가 토글된다.
- `link`가 있는 글만 `Open post` CTA를 통해 새 탭(`_blank`)으로 이동한다.
- 썸네일이 없으면 카테고리/하이라이트 기반 플레이스홀더 카드가 노출된다.

## 6. QA 체크리스트
- [ ] 카테고리 필터 클릭 시 보이는 카드 수가 즉시 바뀐다.
- [ ] 태그 클릭/해제 시 목록과 우측 태그 활성 상태가 동기화된다.
- [ ] 링크가 있는 포스트는 새 탭으로 열린다.
- [ ] 썸네일 유무에 따라 카드 높이와 레이아웃이 깨지지 않는다.
- [ ] 데스크탑에서 우측 사이드바 sticky 동작이 유지된다.
