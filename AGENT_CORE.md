# AGENT CORE

IMPORTANT: 모든 응답과 주석은 한국어로 작성한다.
IMPORTANT: 설명보다 실행을 우선한다. 불확실하면 보수적으로 구현하고 근거를 짧게 남긴다.
중복 설명을 피하고, 바로 실행 가능한 명령형 문장으로 답한다.

작업 전 영향 파일을 식별하고, 고위험 파일이면 변경 범위를 최소화한다.
고위험 파일: `src/lib/googleSheets.ts`, `src/data/translations/**`, `package.json`, `package-lock.json`, `tailwind.config.js`, `.env*`.
고위험 파일은 목적·리스크·검증 결과를 변경 보고에 반드시 포함한다.

존재하지 않는 파일/문서/경로를 규칙에서 참조하지 않는다.
추론 가능한 정보(기술스택 나열, 트리 반복 설명)는 규칙에서 제거한다.
규칙 간 충돌 시 안전성 > 일관성 > 편의성 순서로 우선 적용한다.
탐색/리뷰 기본 제외 경로: `node_modules/**`, `.next/**`, `coverage/**`, `dist/**`.

기본 검증 루프를 항상 수행한다.
1) 변경 직후 정적 점검: `npm run lint`
2) 배포 전 점검: `npm run build`
3) 실패 시 원인 수정 후 동일 순서로 재검증

검증을 생략하면 생략 이유와 대체 확인 방법을 보고한다.
외부 의존성 추가/업데이트는 필요 최소한으로 수행하고 변경 이유를 기록한다.
의존성 변경 시 `package-lock.json` 일관성을 함께 유지한다.

수동 수정 금지 대상: `node_modules/**`, `.next/**`, 빌드 산출물, 자동 생성 파일.
생성물이 필요하면 생성 명령으로 갱신한다.

아키텍처 핵심을 유지한다.
1) UI/페이지 로직은 `src/app/**`, 재사용 UI는 `src/components/**`에 둔다.
2) 번역 키 변경 시 `src/data/translations/**/*.ts`의 다국어 매핑 누락을 금지한다.
3) 클라이언트 훅/상태/애니메이션 사용 컴포넌트는 `'use client'` 누락을 금지한다.
4) 외부 API 연동 변경은 실패 경로와 환경변수 의존성을 함께 점검한다.

문서 기준을 따른다.
코드 동작이 바뀌면 관련 문서를 같은 턴에 업데이트한다.
문서는 설계 배경이 아니라 운영 규칙과 검증 절차 중심으로 짧게 유지한다.
규칙 파일은 공통 코어 1개 + 벤더별 얇은 파일 구조를 유지한다.
