# Styling Rules

스타일 수정은 기존 토큰/유틸 우선으로 처리하고 임시 인라인 스타일 남용을 금지한다.
색상/간격 체계 변경 시 `tailwind.config.js` 정의와 사용처를 동시에 맞춘다.
컴포넌트별 클래스 중복이 3회 이상이면 공통화한다.
접근성 회귀를 막기 위해 대비와 포커스 표시 제거를 금지한다.
시각 변경 커밋에는 변경 전후 검증 근거(린트/빌드 또는 스냅샷)를 포함한다.
