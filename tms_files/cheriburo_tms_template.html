Solution Details

방안 1: 표준 TMS 프로세스 도입 (현재 시스템)
전날 ERP에서 농장·차량·시간 계획을 작성하고 TMS/기사 앱으로 그대로 전달하는 완전 표준형 운영으로, 기존 엑셀 업무를 ERP/TMS 체계로 전환합니다.
1 ERP 계획 작성 (전날)
2 TMS 차량·기사 배정
3 기사 앱 지시
4 실시간 모니터링
✓ 실시간 관제, 표준 기능 100% 활용, 계획/실적 데이터 정합성 확보 ✗ ERP·TMS 중심으로 업무를 재설계해야 하며 기사/관리자 교육과 2~3개월 정착 기간이 필요
고객 부담: ★★★★★
방안 2: TMS를 뷰어로만 활용
고객사는 ERP에만 입력하고, ERP가 제공하는 API를 통해 TMS가 데이터를 읽어 관제/모바일 화면에 그대로 시각화하는 뷰어형 연동 구조입니다.
1 ERP 계획+차량 관리 (전날)
2 ERP-TMS API 연동
3 TMS 관제·앱 화면 노출
✓ ERP 하나만 사용하면서 실시간 관제를 구현, 중복 입력 없음 ✗ ERP-TMS 연동 개발과 ERP 내 차량 관리 기능 정비가 필요하며 ERP 장애 시 관제도 영향을 받음
고객 부담: ★★★☆☆
방안 3: 후행 관제 (ERP 연동)
당일에는 엑셀/수기로 운영하고 종료 후 ERP에 결과를 입력하면, TMS가 해당 데이터를 불러와 운행 경로·타임라인·통계를 복기해 주는 방식입니다.
1 당일 수기 작업
2 종료 후 ERP 입력
3 TMS 지도/타임라인 복기
✓ 업무 방식 변경 없이 ERP 데이터 자산으로 운행 기록을 체계적으로 시각화 ✗ 실시간 관제와 긴급 대응은 불가능하며 ERP 연동 개발 부담이 존재
고객 부담: ★☆☆☆☆
방안 4: 독립형 후행 관제
ERP 연동 없이 전날에 미리 TMS에 차량·농장·출발/도착 시간만 최소 입력하여 독립적으로 운행 복기와 리포트를 제공하는 가장 심플한 방식입니다.
1 당일 수기 작업
2 종료 후 TMS에 핵심 정보 입력
3 지도/타임라인 리포트 생성
✓ 개발 없이 빠르게 도입하고 고객 저항을 최소화하며 시각화 가치를 즉시 제시 ✗ 실시간 관제는 불가하고 최소 입력과 농장/차량 마스터 관리를 TMS에서 수행해야 함
고객 부담: ★★☆☆☆
