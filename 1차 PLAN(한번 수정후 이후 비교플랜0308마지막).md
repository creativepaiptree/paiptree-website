# 메인 홈 비주얼 리듬 복원 계획

## Summary
현재 홈의 문제는 메시지 구조가 아니라 시각 구조다. 구현 방향은 유지하되, 메인 화면을 다시 `강한 히어로 비주얼`, `캐러셀 기반 중간 리듬`, `증거가 읽히는 시각 블록`으로 재구성한다. 목표는 팔란티어처럼 보이는 것이 아니라, 팔란티어급의 시각 밀도와 리듬을 가지면서도 파이프트리 서사에 맞는 첫인상을 만드는 것이다.

## Implementation Changes
### 1. Hero를 텍스트 중심에서 "추상 시스템 히어로"로 전환
- 현재 홈 히어로의 좌측 텍스트 구조는 유지하되, 우측 운영 스택 박스를 대체해 대형 비주얼 스테이지를 만든다.
- 비주얼은 사진형 히어로가 아니라 `추상 시스템 그래픽 + 실제 제품 단서` 혼합 방식으로 설계한다.
- 구성은 3층으로 고정한다: 배경 그리드/라인, 중앙 시스템 다이어그램형 구조, 전면 floating panel 2~3개.
- floating panel 내용은 기존 제품 자산에서 추출한다: `AI 중량/현장`, `EMS 운영 대시`, `TMS 실행 흐름`.
- 패널은 실제 스크린샷을 그대로 크게 노출하지 않고, sharp crop + mono label + thin border 형태로 재구성한다.
- Hero 안에서 CTA 아래에 미니 proof row를 추가한다: `360개 농장`, `30억+ 데이터`, `92%+ 정확도`.
- 메인 이미지의 역할은 “농장 사진”이 아니라 “운영 시스템이 실제로 존재한다”는 신호를 주는 것으로 정의한다.

### 2. 중간 구간에 캐러셀을 복원하되, 제너릭 슬라이드는 폐기
- 예전 `InfiniteCarouselSection`의 UX 패턴은 재사용 가능하지만, 콘텐츠는 전면 교체한다.
- 캐러셀은 5장이 아니라 3장으로 축소한다. 각 장은 제품 레이어 하나에 대응한다: `Farmers Mind AI`, `Farmers Mind EMS`, `Farmers Mind TMS`.
- 각 슬라이드는 다음 구조로 통일한다: 상단 label, 대형 타이틀, 1문장 서브카피, 우측/배경 시스템 비주얼, 하단 짧은 proof line.
- 슬라이드마다 추상 그래픽 언어를 다르게 준다.
  - AI: weight curve, sensing grid, flock heat pattern
  - EMS: dashboard frame, rearing/settlement blocks, control console composition
  - TMS: route line, ETA block, dispatch map abstraction
- 자동 전환은 유지하되 속도는 6초 이상으로 늦추고, 사용자가 hover하면 정지한다.
- 버튼형 탭은 유지하되 현재보다 더 무겁게 보이도록 `number + title + progress line` 구조로 바꾼다.
- 캐러셀의 목적은 “기술 이미지 쇼”가 아니라 “3개 제품 레이어를 한 번에 체험시키는 메인 장치”로 고정한다.

### 3. 섹션 리듬을 텍스트 블록 반복에서 "대비형 구조"로 재배열
- 현재 `Problem`, `Thesis`, `Impact`, `Proof`, `Global Expansion`이 모두 유사한 텍스트 그리드 톤이라 리듬이 평평하다. 이를 4개의 다른 시각 패턴으로 재배치한다.
- `Problem`: 다크 에디토리얼 텍스트 섹션 유지. 첫 문제 정의 섹션은 지금 구조 유지.
- `Thesis`: 현재처럼 텍스트만 두지 말고 우측에 시스템 다이어그램형 정적 비주얼 추가.
- `Product System`: 기존 라이트 리스트형 대신 캐러셀 뒤 보강 섹션으로 축소. 3개 제품의 관계를 보여주는 horizontal architecture strip으로 변환.
- `Operating Impact`: 3컬럼 카드 유지 가능하나 각 카드에 숫자 또는 mini signal bar를 넣어 데이터성 강화.
- `Proof & Traction`: 지금의 3분할 텍스트 카드 대신 `featured proof block + 2개 supporting proof` 구조로 재설계.
- `Global Expansion`: 텍스트 + 숫자 마커 대신 world/route abstraction을 배경으로 둔 지도형 섹션으로 전환.
- `Newsroom`: 현재 리스트 구조는 유지하되, 메인 첫 화면의 비주얼 밀도를 보완하기 위해 카드/슬라이드 성격을 일부 되돌린다.

### 4. 메인 화면 전용 비주얼 자산 체계를 만든다
- 콘텐츠 원본은 계속 `src/content/siteContent.ts`에 두되, 홈 전용 시각 설정은 별도 객체로 분리한다.
- 추가할 내부 인터페이스는 `heroVisual`, `carouselSlides`, `proofSignals`, `visualThemeTokens` 정도로 제한한다.
- 기존 `p1.png ~ p5.png`는 메인 핵심 비주얼에서 제외한다. 제너릭해서 다시 같은 문제가 생긴다.
- 우선 사용할 자산은 현재 리포지토리 내 실제 제품 자산으로 한정한다: `tms_*`, 필요 시 `PoC` 시각 블록 일부, 기존 로고/파트너 자산.
- 새 자산이 필요하면 이미지 추가보다 CSS 기반 추상 구성과 crop 재조합을 우선한다.

### 5. 정보 구조는 유지하고, 메인에서만 체험 밀도를 높인다
- 현재 홈의 메시지 순서는 유지한다: `Hero → Problem → Thesis → Product System → Impact → Proof → Global Expansion → Vision → Partners → News → CTA`.
- 변경 포인트는 순서가 아니라 각 구간의 시각 표현 방식이다.
- `/about`는 지금처럼 회사소개형으로 두고, `/`만 더 공격적인 landing 구성으로 만든다.
- 메인에서 팔란티어식 리듬을 가져오되, `/about`까지 동일 강도로 복제하지 않는다. 루트와 회사소개의 역할을 계속 분리한다.

## Public Interfaces / Types
- 홈 콘텐츠 데이터에 시각 구성을 위한 전용 필드를 추가한다.
- 추천 shape:
  - `homeContent.heroVisual`: 히어로 패널, 라벨, 강조 라인 정보
  - `homeContent.carouselSlides`: 3개 제품 레이어 슬라이드 데이터
  - `homeContent.proofSignals`: 숫자/짧은 proof line
- 기존 페이지 라우트나 외부 API 인터페이스는 변경하지 않는다.
- 뉴스 API 사용 방식은 유지하고, 메인 뉴스 섹션 표현만 바꾼다.

## Test Plan
- 데스크톱 첫 화면에서 텍스트 없이도 제품성/시스템성이 느껴지는지 확인한다.
- Hero에서 headline, visual stage, CTA, mini proof가 한 화면에 안정적으로 공존하는지 확인한다.
- 캐러셀 3장이 자동 전환, hover 정지, 수동 탭 이동 모두 정상 동작하는지 확인한다.
- 모바일에서 히어로 비주얼이 과도하게 잘리지 않고, 캐러셀이 단일 카드 구조로 자연스럽게 축소되는지 확인한다.
- `/`, `/about`, `/services`의 역할 차이가 명확한지 확인한다.
- `npm run lint`, `npm run build` 통과를 유지한다.
- 기존 PoC 경고와 문서 코드블록 경고는 신규 작업 범위 밖으로 유지한다.

## Assumptions
- 사용자가 선택한 방향은 `추상 시스템 기반 히어로`와 `캐러셀 복원`이다.
- 메인 이미지 복원은 농장 사진 복원이 아니라 시스템 비주얼 복원으로 해석한다.
- 캐러셀은 예전 5장 generic card가 아니라 3장 product-system carousel로 재설계한다.
- 새 디자인의 기준은 `docs/web-design-system.md`를 유지하되, 그 안에서 허용된 Hero mesh-gradient와 구조선 언어를 적극적으로 사용한다.
- 구현은 홈(`/`) 중심으로 먼저 수행하고, `/about`는 현재 구조를 유지한 채 필요한 최소한의 톤 정리만 한다.
